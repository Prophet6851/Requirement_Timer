------

# 🧩 一、项目定位 & 目标功能

Requirement Timer 是一款为 **游戏策划** 特别定制的轻量级需求管理工具，使用 Python + PySide6 + SQLite 开发，支持以下目标：

- **快速记录需求并管理推进状态**
- **自动计算推进天数、剩余天数、逾期状态**
- **自动计算需求推进进度条（Apple 扁平风格）**
- **支持归档/取消归档/删除归档任务**
- **支持按模块、状态、优先级筛选任务**
- **提供日历视图（Calendar View），按日期查看任务排期**

UI 风格以 **macOS（Apple 风）** 为核心设计方向。

# 🗂 二、当前完成的全部功能（2025-02）

## ✔ 1. 任务管理（主界面）

每个任务包含字段：

| 字段         | 含义                                 |
| ------------ | ------------------------------------ |
| 标题         | 必填，需求名称                       |
| 模块         | 系统 / AI / 数值 / …                 |
| 版本号       | v2.3.0 等                            |
| 状态         | 策划中 / 开发中 / 待验收 / 已上线 等 |
| 优先级       | 高 / 中 / 低                         |
| 计划开始日期 | YYYY-MM-DD                           |
| 计划截止日期 | YYYY-MM-DD                           |
| 备注         | 自由填写，支持多行文本               |

表格中显示：

- 标题
- 模块
- 状态
- 优先级
- 开始日期 / 截止日期
- 已推进天数
- 剩余天数（含 **已逾期** / **已完成** 自动处理）
- **备注（新增加列）**
- 进度条（扁平 Apple 风格）

系统自动：

- 计算推进天数（start→today）
- 计算剩余天数（today→end）
- 自动显示：
  - 逾期任务：**红色高亮**
  - 完成任务：**绿色“已完成”**
  - 完成任务进度强制 100%

## ✔ 2. 需求归档管理（Archive）

单独弹窗页面，显示所有状态 = “已上线（归档）” 的任务。

操作支持：

- **移出归档**：将任务从归档恢复为 `待验收`
- **删除**：彻底删除归档任务
- 归档页面也显示备注列
- 自动显示“已完成”为绿色

主界面顶部显示统计：

```
当前任务：X / 归档：Y
```

------

## ✔ 3. 新建/编辑任务（独立 TaskDialog）

一个独立弹窗，含：

- 标题输入框
- 模块输入框
- 版本号输入框
- 状态下拉框（含所有状态）
- 优先级下拉框
- 开始日期 & 结束日期（日期选择器）
- 备注编辑框（多行）

填写时自动回显原任务内容。

标题不能为空。

## ✔ 4. 日历视图（Calendar View）⭐ **新功能**

作为独立文件：`calendar_view.py`

从主界面底部按钮进入：

```
日历视图
```

### 功能表现：

- 左侧为可选日期的日历控件（支持网格）
- 右侧显示：
  - 选中日期
  - 当天任务数量
- 下方表格显示"该日期正在推进的所有任务"

### 日期匹配规则：

任务在某天被视为“正在进行”，如果：

```
start_date <= 选中日期 <= end_date
```

并含：

- 状态字段
- 备注字段
- 完成任务文字为绿色
- 自动调整列宽

### 为什么重要？

策划可以通过日历一眼看到：

- 某天的工作量
- 哪些需求在同一日期重叠
- 排期是否堆积

非常实用的节奏管理工具。
