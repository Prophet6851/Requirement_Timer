# **1. 软件简介**

Requirement Timer 是一款为 **游戏策划、系统设计师、产品经理** 开发的轻量级任务推进管理工具，用于：

- 管理需求的不同推进阶段
- 自动计算推进天数、剩余时间与进度
- 支持需求归档与取消归档
- 提供日历视图，查看某天有哪些需求正在进行
- 使用 **本地 SQLite** 保存数据，无需联网
- 支持打包为独立 exe，可在任意电脑直接运行

界面风格简洁，操作直观，适合独立开发者与工作室内部使用。

# **2. 系统要求**

- Windows 10 或 Windows 11
- 无需安装 Python（如果你使用 exe 版本）
- 推荐屏幕分辨率：1920 × 1080 或以上

# **3. 安装与启动**

## 3.1 使用 exe 运行（推荐）

1. 打开打包后生成的文件夹，例如：

   ```
   dist/Requirement Timer/
   ```

2. 双击：

   ```
   Requirement Timer.exe
   ```

3. 如需桌面图标，可右键 → “创建快捷方式”。

# **4. 主界面说明**

主界面由三个主要部分组成：

### **顶部区域（筛选 & 状态统计）**

- 显示当前任务数量与归档数量
- 状态筛选：按任务推进状态过滤
- 优先级筛选：高 / 中 / 低

### **中部区域（任务列表）**

表格包含以下列：

| 列名       | 内容                   |
| ---------- | ---------------------- |
| 标题       | 需求名称               |
| 模块       | 系统/AI/数值等         |
| 版本       | 需求所属版本号         |
| 状态       | 当前推进阶段           |
| 优先级     | 高/中/低               |
| 开始日期   | 计划开始时间           |
| 截止日期   | 计划结束时间           |
| 已推进天数 | 从开始到今天的天数     |
| 剩余天数   | 距离结束日期的剩余天数 |
| 备注       | 额外信息               |
| 进度       | 自动计算的完成度       |

### **底部按钮区**

包含操作按钮：

- **新建**
- **编辑**
- **删除**
- **刷新**
- **查看归档**
- **日历视图**
- **关于本软件**

# **5. 功能说明**

## **5.1 新建任务**

点击 **“新建”** → 新建任务窗口会弹出。

需填写内容：

- 标题（必填）
- 模块
- 版本号
- 状态
- 优先级
- 开始日期
- 结束日期
- 备注

点击“确定”保存。

------

## **5.2 编辑任务**

- 在表格中选中任务 → 点击“编辑”
- 所有信息均可修改
- 点击“确定”保存修改

------

## **5.3 删除任务**

- 选中要删除的任务 → “删除”
- 删除后不可恢复（包括归档任务），请谨慎操作

------

## **5.4 查看归档任务**

点击 **“查看归档”** 会打开独立的归档窗口：

在归档中可进行：

| 功能     | 说明                 |
| -------- | -------------------- |
| 移出归档 | 将任务恢复为“待验收” |
| 删除     | 彻底删除任务         |

归档中任务的进度和状态自动视为“已完成”。

------

## **5.5 日历视图**

点击 **“日历视图”** 打开一个独立的日历排期窗口。

左侧：日历
 右侧：选中日期信息
 下方：选中日期正在进行的任务列表

任务被视为在某日进行的规则：

```
如果 start_date ≤ 选中日期 ≤ end_date，则任务属于该日
```

用于查看排期密度、工作量集中情况等。

------

## **5.6 筛选功能**

主界面左上角提供两个筛选：

- **状态筛选**：不包含“已上线（归档）”
- **优先级筛选**：高 / 中 / 低 / 全部

修改筛选几乎瞬时刷新表格。

------

## **5.7 进度与天数计算逻辑**

### **推进天数（已推进天数）**

```
today - start_date，最少为 0
```

### **剩余天数**

```
end_date - today（可为负数）
```

### **状态显示规则**

- 状态 ∈ {验收完, 测试回归bug, 已上线（归档）} → **已完成（绿色）**
- 剩余天数 < 0 且未完成 → **已逾期（红色）**

### **进度条计算**

```
progress = spent / (spent + left)
```

特殊情况：

- 完成状态 → 强制 100%
- 逾期 → 强制 100%
- 时间数据缺失 → 空白

# **6. 注意事项**

- 删除任务不可恢复，请谨慎操作
- 程序运行目录必须有写入权限（SQLite 需写入）
- 未填写日期的任务不显示在日历视图中
- 推荐在执行大量操作之前备份数据库文件 `task.db`

# **7. 常见问题（FAQ）**

### Q1：程序关闭后数据会丢失吗？

不会。所有数据保存在 `task.db` 中。

### Q2：换电脑后如何迁移数据？

复制 `task.db` 到另一台电脑即可。

### Q3：归档任务能否重新放回？

可以，通过“移出归档”恢复为 `待验收`。

### Q4：日历里为什么看不到某些任务？

任务需要正确设置开始和截止日期。
